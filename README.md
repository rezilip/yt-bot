# ربات دانلود یوتیوب — دیپلوی رایگان روی Render

## ⚠️ اول از همه: امنیت توکن
توکنی که تو چت فرستادی از این لحظه لو رفته حساب می‌شه. همین الان:
1. برو به [@BotFather](https://t.me/BotFather)
2. `/mybots` → ربات خودتو انتخاب کن → `API Token` → `Revoke current token`
3. توکن جدید رو فقط داخل Environment Variables خود Render بذار (پایین توضیح دادم)، هیچ‌وقت داخل کد ننویسش.

## ساختار پروژه
```
app.py          Flask + اجرای ربات در ترد پس‌زمینه (برای بیدار موندن رایگان روی Render)
bot.py          تمام handlerها، منو، قفل کانال، پنل ادمین
db.py           دیتابیس SQLite (کاربران، VIP، بن، سهمیه‌ی روزانه)
i18n.py         متن‌های ۴ زبانه (فارسی، انگلیسی، عربی، روسی)
downloader.py   دانلود از یوتیوب با yt-dlp (محدودیت ۵۰۰ مگابایت)
Dockerfile      شامل نصب ffmpeg (لازم برای تبدیل به MP3)
requirements.txt
.env.example
```

## مراحل دیپلوی روی Render (رایگان)

### ۱. ساخت ریپازیتوری در گیت‌هاب
پروژه رو در یه ریپازیتوری گیت‌هاب (می‌تونه private باشه) آپلود کن. فایل `.env` واقعی رو **هرگز** push نکن — فقط `.env.example` باید توی ریپو باشه.

### ۲. ساخت سرویس در Render
1. وارد [render.com](https://render.com) شو و با گیت‌هاب لاگین کن.
2. `New +` → `Web Service` → ریپازیتوری پروژه رو انتخاب کن.
3. چون `Dockerfile` داری، Render خودش تشخیص می‌ده و ازش استفاده می‌کنه (نیازی به تنظیم Build/Start Command نیست).
4. پلن `Free` رو انتخاب کن.

### ۳. تنظیم Environment Variables
تو تنظیمات سرویس، بخش `Environment` این مقادیر رو اضافه کن:
```
BOT_TOKEN = <توکن جدیدی که از BotFather گرفتی>
ADMIN_IDS = 5997515450
CHANNEL_USERNAME = @GetYTMedia
CHANNEL_LINK = https://t.me/GetYTMedia
FREE_DAILY_LIMIT = 5
MAX_FILE_MB = 500
SUPPORT_USERNAME = @irezafattahi
```
(آدرس‌های ولت داخل کد `i18n.py` هاردکد شدن و نیازی به env نیست، ولی اگه خواستی عوضشون کنی همون فایل رو ویرایش کن.)

### ۴. Deploy
دکمه‌ی `Create Web Service` رو بزن. Render شروع به build کردن Docker image می‌کنه (چند دقیقه طول می‌کشه چون ffmpeg هم نصب می‌شه). بعد از اتمام، لاگ‌ها رو چک کن — باید ببینی ربات polling رو شروع کرده.

### ۵. جلوگیری از خواب رفتن (مهم!)
سرویس‌های رایگان Render بعد از ۱۵ دقیقه بدون ترافیک HTTP می‌خوابن. برای بیدار نگه‌داشتنش:
1. برو به [cron-job.org](https://cron-job.org) (رایگان) و ثبت‌نام کن.
2. یه cron job جدید بساز که هر ۱۰ دقیقه به آدرس سرویس Renderت (مثلاً `https://your-app.onrender.com`) درخواست GET بزنه.
3. این باعث می‌شه Render سرویس رو زنده نگه داره و ربات همیشه در حال polling باشه.

### ۶. ربات رو در کانال ادمین کن
حتماً ربات باید تو کانال `@GetYTMedia` عضو و **ادمین** باشه (برای اینکه بتونه وضعیت عضویت کاربرا رو چک کنه).

### ۷. تست
تو تلگرام `/start` بزن؛ باید انتخاب زبان بیاد، بعد قفل کانال، بعد منوی اصلی با چیدمون دکمه‌هایی که خواسته بودی:
```
        ⬇️ دانلود ویدیو
👑 VIP           🆘 پشتیبانی
💎 دونیت          📖 راهنما
      🌐 تغییر زبان
```

## دستورات ادمین (فقط برای آیدی 5997515450)
- `/admin` — نمایش لیست دستورات
- `/stats` — آمار کاربران
- `/vip <آیدی>` / `/unvip <آیدی>` — VIP کردن / حذف VIP
- `/ban <آیدی>` / `/unban <آیدی>` — مسدود کردن / رفع مسدودی
- `/broadcast <متن>` — پیام همگانی به همه‌ی کاربران

## محدودیت‌های مهمی که باید بدونی
- **دیسک Render رایگان موقتیه (ephemeral)** — فایل دیتابیس SQLite (`bot.db`) با هر ری‌استارت سرویس (که گاهی خودکار اتفاق می‌افته) از بین می‌ره و آمار/VIPها ریست می‌شن. برای پایدار موندن داده‌ها، بعداً باید یه دیتابیس خارجی رایگان مثل [Supabase](https://supabase.com) یا [Neon](https://neon.tech) (پستگرس رایگان) وصل کنی. اگه خواستی همین الان اینو راه بندازیم بگو.
- دانلود ویدیوهای خیلی طولانی یا با کیفیت بالا ممکنه با محدودیت CPU/RAM پلن رایگان Render کند یا fail بشه.
- دانلود محتوای دارای کپی‌رایت باید مطابق قوانین یوتیوب و کشور محل استفاده باشه؛ مسئولیت استفاده از محتوای دانلودشده با کاربرانه.
